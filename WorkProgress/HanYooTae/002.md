# DB 설계

## Table은 아래와 같을 예정.

1. 기본 능력의 부모격인 AbilityBase
2. AbilityBase에서 파생되는 능력들
3. Character

### AbilityBase Column
1. abilityName(능력 이름)
2. description(능력 설명)

### AbilityBase에서 파생되는 능력들의 Column
1. 각 능력 Table들은 모두 각자 능력의 고유 flag를 가질 예정  ex) CardBonusAbility -> enableCardBonus

### Character Column
1. abilityName
2. 능력을 사용할 수 있는지 여부
3. 능력을 사용했는지 여부
4. 1 ~ 4까지의 카드 존재 여부 // 1 ~ 4까지의 Column이 독립적으로 있으며, 0이면 없고, 1이면 1장, 2면 2장 가지고 있게 만들면 될듯? flag가 아닌 int형으로 만든 이유는 Car
5. 플레이어가 현재 위치하고 있는 칸
